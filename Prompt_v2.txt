I. STRUCTURAL & TECHNICAL REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Technology: Clean semantic HTML5, modern CSS (Flexbox/Grid, CSS variables), vanilla JS for
interactions only where necessary.

Output files (MINIMUM — do not combine into one file):
  - index.html      (Home page)
  - services.html   (Full Services page)
  - team.html       (Our Team page)
  - contact.html    (Contact page with form)
  - success.html    (Form submission confirmation page)
  - styles.css      (Global shared stylesheet)
  - script.js       (Shared JS for interactivity)

Design System (CRITICAL):
  Buttons: Pill-shaped (border-radius: 50px), solid teal background, white text,
           hover lift (transform: translateY(-2px)), no gradients on default.
  Cards:   White background, 1px solid #E5E5E5, border-radius: 16px,
           subtle box-shadow: 0 4px 6px rgba(0,0,0,0.05), clean padding.
  Inputs:  border-radius: 12px (NOT the full 50px pill), light gray border,
           focus state with teal border, no outline.
  Spacing: Generous whitespace. Sections separated by 80-120px. No cramped layouts.

Performance (CRITICAL — affects PageSpeed score):
  - Load Google Fonts via HTML <link> with rel="preconnect" (NOT @import in CSS).
  - All images must have explicit width and height attributes to prevent layout shift (CLS).
  - Do NOT use background-attachment: fixed — this kills mobile GPU performance.
    Use background-attachment: scroll instead.
  - Lazy-load all below-the-fold images (loading="lazy").
  - Hero background image should use a teal gradient overlay + background image URL.

SEO (ALL pages must include):
  - <meta name="description"> — unique, keyword-rich, max 160 characters.
  - <meta name="robots" content="index, follow">
  - <link rel="canonical" href="...">
  - Open Graph tags: og:title, og:description, og:type, og:url
  - Semantic HTML5 structure: one <h1> per page, with proper h2/h3 hierarchy.
  - Descriptive alt text on all images.

Logo (IMPORTANT — detailed spec):
  Do NOT use a dental tooth emoji or a regular emoji circle face in the logo.
  The logo should be the wordmark "SmileCare" (text only), styled as two parts:
    - "Smile" in the primary teal color
    - "Care" in the accent teal color
  Below the text, place an Amazon-style curved SVG smile arrow that:
    - Spans the full width of the wordmark
    - Curves gently from left to right with a small arrowhead tip on the right end
    - Is absolutely positioned below the text using position: relative on the anchor
      and position: absolute; bottom: 0; left: 0; width: 100% on the SVG
    - Uses the accent teal color (#14A098)
  Example SVG for the smile: viewBox="0 0 100 24"
    <path d="M 5,8 Q 50,28 95,8" stroke="currentColor" .../>
    <path d="M 85,3 L 95,8 L 88,17" stroke="currentColor" .../> (arrowhead)
  The footer logo uses the same structure but inherits white color.

Navigation Links:
  Home | Services | Our Team | Patient Reviews | Contact
  Active page link should be highlighted with teal color.
  On mobile: hamburger menu toggle with smooth max-height + opacity CSS transition animation
             (NOT display:none/flex toggle — that has no animation).

CTA (top right): "Book Appointment" — pill button, teal, white text.
  On mobile: hide the CTA button, only show hamburger toggle.

XI. CRITICAL RULES — DO NOT VIOLATE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✗ NO gradient buttons
✗ NO heavy box shadows on everything
✗ NO serif fonts
✗ NO carousel/slider in hero
✗ NO hamburger menu visible on desktop
✗ NO more than 2 font sizes per section
✗ NO decorative background patterns
✗ NO background-attachment: fixed (kills mobile performance — use scroll)
✗ NO @import for fonts in CSS (use HTML <link> with preconnect instead)
✗ NO hardcoded year in copyright — use JavaScript Date()
✗ NO emoji or generic icon in the logo — use the specified SVG text + curve design
✗ DO NOT use display:none/flex to toggle the mobile menu — use max-height + opacity transition
✗ DO NOT link "Contact" menu items to index.html#contact — always use contact.html
